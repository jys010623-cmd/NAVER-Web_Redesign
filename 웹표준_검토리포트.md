# NAVER 리디자인 — 웹 표준 검토 리포트

검토 대상: `NAVER/` (index, login, join, sub01_05, header, style.css, action.js 외)
기준: 대기업 실무 수준의 웹 표준 / 접근성(WCAG·KWCAG) / SEO / 성능 / 코드 품질

전반적으로 마크업·반응형 구조는 잘 잡혀 있으나, **실제 동작하지 않는 JS 버그**와 **접근성·SEO 미흡**, **다른 회사 잔재(Zinus)**, **존재하지 않는 링크**가 실무 기준에서 가장 큰 감점 요소입니다. 우선순위 순으로 정리했습니다.

---

## 1. 치명적 (동작 불가 / 즉시 수정)

### 1-1. JS 선택자 오타로 플로팅 메뉴 기능 전체 미작동
`script/action.js`에서 `floating_menu`를 클래스가 아닌 **태그 선택자**로 쓰고 있어 아무것도 선택되지 않습니다.

```js
$('floating_menu li').eq(0)...   // ❌ <floating_menu> 태그를 찾음
$('floating_menu a').click(...)  // ❌ 클릭 이벤트가 안 붙음
```
→ `'.floating_menu li'`, `'.floating_menu a'` 로 수정해야 스크롤 활성표시·클릭 이동이 동작합니다. (238, 241, 244, 263행)

### 1-2. 정의되지 않은 변수로 스크롤 시 에러
```js
if(scrT > part3Top - win3H/3){   // ❌ win3H 미정의 → ReferenceError
```
`win3H` → `winH` 오타. 스크롤할 때마다 콘솔 에러가 발생하고 이후 코드가 중단됩니다. (243행)

### 1-3. 헤더에 다른 회사명·깨진 링크 잔재
`include/header.html`에 `data-en="ABOUT ZINUS"` (Zinus, 침대 회사) 가 남아 있고, 메뉴가 **존재하지 않는 페이지**(`sub01_01~04.html`, `sub03_01~05.html`)로 연결됩니다. 실제로 존재하는 파일은 `sub01_05.html` 하나뿐입니다.
또한 `SUBMENU1~8`, "비젼"(→ 비전) 같은 플레이스홀더·오타가 그대로 노출됩니다. 대기업 사이트라면 깨진 링크(404)는 즉시 감점 요소입니다.

### 1-4. HTML 구조 오류 (join.html)
성별 선택 영역의 `<div class="content align_garo">`가 **닫히지 않은 채** `</li>`가 옵니다. (80~83행) 태그 짝이 맞지 않아 검증(W3C Validator) 실패합니다.

## 2. 접근성 (KWCAG / WCAG)

- **대체텍스트 누락**: 로고·소개 이미지의 `alt=""`가 비어 있습니다(`header.html` 로고, `index.html` `bigimg`). 의미 있는 이미지는 `alt="네이버"`처럼 내용을 기술해야 합니다. (장식 이미지만 빈 alt 허용)
- **폼 레이블 미연결**: `join.html`의 대부분 `<label for="">`가 비어 있고, `login.html`·검색창 input에는 `<label>` 자체가 없습니다. 스크린리더가 입력 항목을 읽지 못합니다. `id`–`for`를 1:1로 연결하세요.
- **빈 의미 요소**: 히어로의 `<strong></strong>`, `<p></p>`가 내용 없이 비어 있습니다. 불필요하면 제거하거나 실제 카피를 넣으세요.
- **아이콘 버튼 레이블**: 검색·슬라이더 버튼이 텍스트("search", "arrow_back_ios")만 있고 `aria-label`이 없습니다. Material Symbols 글자가 그대로 읽힐 수 있어 `aria-label="검색"`, `aria-hidden` 처리가 필요합니다.
- **키보드 포커스/명도대비**: 흰색 GNB 텍스트(`#fff`)가 영상 위에 올라가 배경에 따라 대비가 4.5:1 미만이 될 수 있습니다. 그라데이션 딤 처리 권장.
- **언어/제목 구조**: `lang="ko"`는 잘 지정됨. 단, 페이지마다 `<h1>`(로고)과 섹션 제목 위계는 점검 필요.

## 3. SEO / 메타

- 모든 페이지 `<title>`이 똑같이 **"NAVER"** 입니다. 페이지별 고유 제목(예: "로그인 | NAVER", "회원가입 | NAVER")로 구분하세요.
- `<meta name="description">` 없음 → 검색 스니펫 품질 저하.
- **Open Graph / Twitter Card 없음** → SNS 공유 시 썸네일·설명이 안 나옵니다. 대기업 필수 항목.
- **favicon 없음**, `robots.txt`·`sitemap.xml` 없음.
- 의미적 마크업: 히어로가 `<div id="hero">`입니다. `<section>`/`<main>` 등 랜드마크 태그 사용 시 SEO·접근성에 유리합니다. 본문 전체를 `<main>`으로 감싸는 것을 권장합니다.

## 4. 성능

- **autoplay 영상 3개**가 첫 화면에서 동시에 로드·재생됩니다. 모바일·저사양에서 부담이 큽니다. `preload="none"`, 포스터 이미지, 또는 첫 슬라이드만 재생 후 lazy 처리 권장.
- 이미지에 `loading="lazy"`, `width`/`height`(CLS 방지), `srcset`(반응형) 미적용.
- **Swiper 초기화 스크립트가 모든 HTML에 인라인 중복** 복붙되어 있습니다(login·join·sub 페이지엔 Swiper 슬라이드가 없는데도 포함). `action.js`로 일원화하세요.
- 외부 CDN(jsdelivr, daum map, google fonts) 의존이 많고 **SRI(integrity) 해시가 없습니다**. 보안·안정성상 핵심 라이브러리는 로컬 호스팅 또는 SRI 적용 권장.
- jQuery 파일명이 `jquery-4.0.0.min.js`입니다. 버전·무결성이 정식 릴리스인지 확인하고, 신규 프로젝트라면 jQuery 의존을 줄이는 방향(Vanilla JS)도 검토하세요.

## 5. 코드 품질 / 유지보수

- **주석 처리된 학습용 코드/메모가 대량으로 운영 파일에 남아 있습니다** (`action.js`의 구구단·jQuery 메모, `join.html`의 음식·학년·파일 input, 각 HTML 하단 Swiper 주석). 배포본에서는 모두 제거해야 합니다.
- `@media screen and(max-width:860px)` — `and`와 `(` 사이 **공백 누락으로 미디어쿼리가 무효**가 됩니다. `and (max-width:860px)`로 수정. (`style.css`) → CSS 변수 반응형 재정의가 적용되지 않고 있습니다.
- 빈 CSS 규칙(`header .topset nav{}`, `.btn_arrow:hover:after{}` 등)·인라인 한글 메모가 다수입니다.
- `var swiper` 대신 `const`/`let`, 전역 변수 최소화 권장.
- 폼 `action`/`method`가 비어 있어 제출 동작이 정의되지 않았습니다(데모라면 무방하나 실무 기준 미완).
- 파일/클래스 네이밍에 한글·임시명(`merong`, `liNum`) 혼재 → 협업 시 컨벤션 정리 필요.
